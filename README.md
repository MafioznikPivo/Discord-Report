# Discord Report

Discord-бот на `disnake` для модерации сервера с двумя основными сценариями:

- `/report` для подачи жалобы на участника.
- `/помощь` для обращения к модераторам через тикет с продолжением диалога в личных сообщениях.

Бот хранит состояние в SQLite, восстанавливает кнопки после рестарта и автоматически закрывает жалобы, если участники не выходят на связь в заданный срок.

## Возможности

- Создание жалоб через slash-команду и модальное окно.
- Проверка, что инициатор и нарушитель состоят на целевом сервере.
- Один активный репорт на пользователя.
- Канал модерации для входящих жалоб с кнопками принять или отклонить.
- Автоматическое создание текстового и голосового каналов для разбора жалобы.
- Перемещение инициатора и нарушителя в voice-канал разбора.
- Автоматическое закрытие жалобы при неявке по дедлайну.
- Система вопросов `/помощь` с ответами модераторов и продолжением переписки через ЛС.
- Восстановление persistent views после перезапуска бота.

## Сценарии работы

### `/report`

1. Пользователь вызывает `/report`.
2. Бот открывает модальное окно с ID нарушителя и причиной жалобы.
3. Жалоба уходит в модераторский канал.
4. Модератор принимает или отклоняет жалобу.
5. При принятии создаются отдельные текстовый и голосовой каналы `report-<id>`.
6. Модератор может переместить инициатора и нарушителя в голосовой канал или закрыть жалобу.
7. Если участник не заходит в voice вовремя, жалоба закрывается автоматически.

### `/помощь`

1. Пользователь вызывает `/помощь`.
2. Бот открывает модальное окно с текстом вопроса.
3. Вопрос появляется в модераторском канале.
4. Модератор может ответить пользователю или закрыть тикет.
5. Пользователь может продолжить диалог в личных сообщениях с ботом, а новые сообщения будут пробрасываться в intake-канал.

## Архитектура

- `bot.py` — точка входа, настройка бота, базы, сервисов, recovery и scheduler.
- `src/config.py` — чтение и валидация переменных окружения.
- `src/cogs/` — slash-команды, модальные окна и обработчики кнопок.
- `src/services/` — бизнес-логика жалоб, help-тикетов, восстановления и дедлайнов.
- `src/db/database.py` — инициализация SQLite и базовые методы доступа к данным.
- `src/ui/views.py` — persistent views и кнопки для жалоб и help-тикетов.
- `data/` — рабочая SQLite-база во время запуска.
- `logs/` — лог-файлы во время запуска.

## Требования

- Python 3.11+
- Discord-бот с включёнными intent'ами:
- `Server Members Intent`
- `Message Content Intent`
- правами на работу с каналами и перемещение пользователей в voice при разборе жалоб

## Установка

### Windows

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
Copy-Item .env.example .env
```

После этого заполните `.env` и запустите:

```powershell
python bot.py
```

### Linux / macOS

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
cp .env.example .env
```

Затем:

```bash
python bot.py
```

Или через подготовленный скрипт:

```bash
chmod +x start.sh
./start.sh
```

## Настройка `.env`

| Переменная | Описание |
| --- | --- |
| `BOT_TOKEN` | Токен Discord-бота |
| `TARGET_GUILD_ID` | ID сервера, на котором работает бот |
| `REPORT_INTAKE_CHANNEL_ID` | Канал, куда отправляются новые жалобы |
| `HELP_INTAKE_CHANNEL_ID` | Канал, куда отправляются новые вопросы `/помощь` |
| `REPORT_CATEGORY_ID` | Категория, в которой создаются каналы разбора жалоб |
| `MODERATOR_ROLE_IDS` | Список ID ролей модераторов через запятую |
| `REPORT_INITIAL_JOIN_DEADLINE_SEC` | Время на первый заход инициатора в voice после принятия жалобы |
| `REPORT_MISSING_MOVE_DEADLINE_SEC` | Время на заход участника в voice после неудачного перемещения |
| `SCHEDULER_POLL_SEC` | Период опроса дедлайнов планировщиком |
| `DB_PATH` | Путь до SQLite-базы |

Пример безопасного значения:

```env
BOT_TOKEN=your_discord_bot_token
TARGET_GUILD_ID=123456789012345678
REPORT_INTAKE_CHANNEL_ID=123456789012345678
HELP_INTAKE_CHANNEL_ID=123456789012345678
REPORT_CATEGORY_ID=123456789012345678
MODERATOR_ROLE_IDS=123456789012345678,234567890123456789
REPORT_INITIAL_JOIN_DEADLINE_SEC=180
REPORT_MISSING_MOVE_DEADLINE_SEC=300
SCHEDULER_POLL_SEC=30
DB_PATH=data/report_sataki.db
```

## Хранение данных

SQLite создаёт таблицы:

- `reports` — жалобы, их статусы, каналы, дедлайны и модераторы.
- `help_tickets` — вопросы пользователей и статус тикета.
- `help_messages` — история сообщений внутри help-тикета.

## Ограничения и замечания

- Бот работает только на одном целевом сервере, заданном в `TARGET_GUILD_ID`.
- Для перемещения пользователей в voice у бота должны быть соответствующие права.
- Если личные сообщения пользователя закрыты, бот пишет системное уведомление в intake-канал.
- Рабочие файлы `.env`, `data/*.db` и `logs/*.log` не должны попадать в публичный репозиторий.
- Если токен когда-либо оказался в репозитории или шаблоне конфигурации, его нужно перевыпустить в Discord Developer Portal.
